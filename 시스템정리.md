## 3. Cloudflare Worker 구조

### 환경변수 (Settings > Variables, 시크릿은 Encrypt ON)

| 변수 | 설명 |
|------|------|
| `GRAPH_CLIENT_ID` / `GRAPH_TENANT_ID` | Azure 앱 등록 정보 |
| `GRAPH_CLIENT_SECRET` ⭐ | Graph API 인증 시크릿 |
| `MEMBER_EMAILS` | `{"이름":"email@..."}` JSON |
| `MEMBER_IDS` | `{"이름":숫자}` JSON |
| `NOTION_SECRET` ⭐ | Notion 통합 토큰 |
| `SCHEDULES_DB_ID` | Notion 일정 DB ID |
| `SPINLOG_DB_ID` | Notion 스핀 이력 DB ID |
| `VTIGER_URL` | vtiger 서버 주소 |
| `VTIGER_USERNAME` / `VTIGER_ACCESS_KEY` ⭐ | vtiger 인증 |
| `VTIGER_MEMBER_IDS` | `{"이름":"19x숫자"}` vtiger 유저 ID |

### Worker 액션 목록 (X-Action 헤더로 구분)

**아웃룩 관련**
- `sync-outlook` — Graph API `calendarView`로 ±4주 일정 가져오기
- `run-sync` — Cron과 동일한 아웃룩→Notion 자동 동기화 수동 트리거
- `create-outlook-event` — 배포 담당자 아웃룩에 `[배포지원 결정]` 일정 생성
- `list-calendars` — 멤버별 캘린더 접근 테스트

**Notion 일정 CRUD**
- `load-events` — SCHEDULES_DB 전체 조회
- `save-event` — 일정 생성(pageId 없음) / 수정(pageId 있음)
- `delete-event` — 일정 아카이브 (삭제)

**스핀 이력**
- `add-spinlog` — SPINLOG_DB에 스핀 기록 추가, Notion pageId 반환
- `get-spinlog` — 스핀 이력 목록 조회 (created_time 내림차순)
- `delete-spinlog` — 스핀 기록 삭제 (배포 취소 시 우선순위 복구용)

**vtiger CRM**
- `vtiger-get-ticket` — 티켓 번호로 티켓 정보 조회
- `vtiger-update-ticket` — 티켓 담당자 변경
- `vtiger-list-users` — vtiger 유저 목록 조회 (VTIGER_MEMBER_IDS 설정 참고용)
- `vtiger-test` — vtiger 연결 상태 디버그

### Cron 자동 동기화

```
wrangler.toml:
crons = ["0 22 * * *", "30 3 * * *", "0 6 * * *"]
       → KST 07:00   → KST 12:30   → KST 15:00
```

`runScheduledSync()` 실행 흐름:
1. Graph API로 ±4주 전체 멤버 아웃룩 일정 수집
2. Notion SCHEDULES_DB에서 기존 아웃룩 이벤트 조회
3. 신규/변경 → upsert, 삭제된 것 → archive

### 아웃룩 이벤트 분류 로직 (cloudflare-worker.js)

```javascript
const OUT_KEYWORDS    = ['연차','반차','휴가','외근','출장','재택','공휴일','경조','병가','조퇴'];
const DEPLOY_KEYWORDS = ['배포','deploy'];
const MEETING_EXCEPTIONS = ['배포회의','[내부업무]','[내부]'];

function classifyEventType(showAs, title) {
  if (showAs === 'oof' || showAs === 'workingElsewhere') return 'out';
  if (OUT_KEYWORDS.some(k => title.includes(k))) return 'out';
  if (MEETING_EXCEPTIONS.some(k => title.includes(k))) return 'meeting';
  if (DEPLOY_KEYWORDS.some(k => title.includes(k))) return 'deploy';
  return 'meeting';
}
```

**색상 매핑:** `meeting` → 🔵 파랑 / `deploy` → 🔴 빨강 / `out` → ⚫ 검정 / `other` → 🟤 갈색

### 배포 가능 인원 판별 로직 (index.html)

```javascript
const FREE_CATEGORIES = ['[티켓]', '[파트너]'];

function isFreeTimeEvent(e) {
  const title = e.title || '';
  const hasFreeCategory = FREE_CATEGORIES.some(cat => title.includes(cat));
  if (!hasFreeCategory) return false;

  // 하루종일 + [티켓]/[파트너] → 배포 가능 (티켓 처리 중이어도 현장 배포 가능)
  if (e.isAllDay) return true;

  // 시간 단위 + '원격' 포함 → 배포 불가 (원격 지원 중)
  if (title.includes('원격')) return false;

  // 시간 단위 + [티켓]/[파트너] + 원격 없음 → 배포 가능
  return true;
}

function isMemberAvailable(memberId, dateStr, startTime, endTime) {
  return !events.some(e =>
    e.memberId === memberId &&
    e.date === dateStr &&
    eventsOverlap(startTime, endTime, e.startTime, e.endTime) &&
    !isFreeTimeEvent(e)
  );
}
```

**판별 규칙 요약:**

| 일정 유형 | 처리 |
|-----------|------|
| OOF/부재중 (아웃룩 showAs) | ❌ 불가 |
| 연차·반차·휴가 등 키워드 | ❌ 불가 |
| 하루종일 + `[티켓]`/`[파트너]` | ✅ 가능 |
| 시간단위 + `[티켓]`/`[파트너]` + 원격 없음 | ✅ 가능 |
| 시간단위 + `[티켓]`/`[파트너]` + 원격 포함 | ❌ 불가 |
| `[신한카드]` 등 고객사 하루종일 일정 | ❌ 불가 |
| 일반 회의/미팅 | ❌ 불가 |

## 5. 라운드로빈 알고리즘

### 우선순위 원리
- `rrOrder` 배열: `[0]` = 다음 차례 (가장 오래 안 뽑힌), `[마지막]` = 최근 선정
- 앱 시작 시 Notion SPINLOG_DB의 `created_time` 기준으로 재구성
- 첫 사용 시 멤버 등록 순서가 기본값

### 스핀 후 처리 흐름

```
돌림판 실행
  └─▶ 가용 멤버만 필터 (기간+시간 기준)
  └─▶ 당첨자 결정
  └─▶ advanceRrOrder(winner) → rrOrder 맨 뒤로 이동
  └─▶ add-spinlog → Notion SPINLOG_DB 기록 + pageId 반환
  └─▶ lastSpinLogPageId 저장

배포 투입 확정
  └─▶ 배포 이벤트 생성 (spinLogPageId 첨부)
  └─▶ create-outlook-event → 담당자 아웃룩에 [배포지원 결정] 일정 생성
```

### 배포 취소/변경 시 우선순위 자동 복구

```
배포 일정 삭제
  └─▶ delete-spinlog(spinLogPageId) → Notion 스핀 기록 삭제
  └─▶ loadRrOrderFromNotion() → 순서 재계산
  └─▶ 해당 인원 우선순위 복구

담당자 변경 (A → B)
  └─▶ delete-spinlog(A의 pageId) → A 우선순위 복구
  └─▶ add-spinlog(B) → B를 후순위로 이동
  └─▶ advanceRrOrder(B)
```

## 7. vtiger CRM 연동

### 동작 위치
- 룰렛 당첨 직후 → 돌림판 모달에 CRM 섹션 표시
- 배포 투입 모달 → CRM 섹션 항상 표시
- 달력의 배포(🔴) 일정 클릭 → 상세 모달에 CRM 섹션 표시

### 인증 방식
vtiger REST API v1 — Challenge/Login (MD5 해시)
```
GET /webservice.php?operation=getchallenge&username=...
  └─▶ token 수신
POST /webservice.php
  └─▶ accessKey = MD5(token + VTIGER_ACCESS_KEY)
  └─▶ sessionName 수신 → 이후 API 호출에 사용
```

> **현재 이슈:** vtiger 서버(crm.softcamp.co.kr)가 Cloudflare Workers IP를 차단 중. nginx에서 Cloudflare IP 대역 허용 필요.
