# 본부 공지 — 배포지원 시스템 아웃룩 태깅 규칙 (최종 메일 본문)

> 아래 내용을 그대로 Outlook 메일에 붙여넣어 발송하시면 됩니다.
> 수신: 커뮤니케이션본부 엔지니어 부서 전 직원 / 참조: 각 부서 부장·팀장

**제목:** `[업무공지] 배포지원 시스템 — 아웃룩 일정 태깅 규칙 안내`

---

안녕하세요, 파트너프로그램팀 이하영입니다.

배포지원 담당자를 공정하게 자동 선정하기 위해, 아웃룩 일정에 '태그'를 다는 규칙을 안내드립니다.
시스템이 일정 제목의 **[태그]** 를 보고 배포지원 투입 가능 여부를 판단하므로, 아래 규칙대로 등록 부탁드립니다.
( 대시보드: https://emitlight.github.io/deploy-schedule/ )

### ■ 핵심 원칙
"일정이 있다 = 무조건 바쁨"이 아니라, 일정 제목의 **[태그]로 '빼도 되는 일'인지** 판단합니다.
→ 배포지원에 빠져도 되는 일은 **반드시 아래 🟢태그**를 달아주세요.

### ■ 일정 태깅 규칙 (제목에 대괄호 [ ] 로 표기)

**🟢 배포지원 "가능" — 진행 중이지만 빼도 되는 일**
`[티켓]` `[파트너]` `[내부]` `[교육]` `[대기]`
- 예) `[티켓] TT119950 암호화문서 오류 대응`
- 예) `[내부] SHIELD Gate 숏폼 제작`
- 예) `[파트너] 인도네시아 PoC 사전준비`
- 예) `[교육] 사내 보안교육`

**🔴 배포지원 "불가" — 빼면 안 되는 일**
`[고객사]` `[외근]` `[출장]` · 연차 / 반차 / 휴가 · **그리고 🟢가능 키워드가 없는 모든 `[태그]`**(고객사명·프로젝트명 등)
- 예) `[고객사] OO사 현장지원` · `[외근] 고객 미팅` · `연차` · `[국민카드] 구축` · `[매그나칩반도체] 지원`
- 즉, 대괄호 `[ ]`를 쓸 거면 **반드시 🟢가능 키워드를 넣어야** 가능으로 잡힙니다. (안 넣으면 전담 업무로 보고 제외)

**⚠️ 급해서 빼면 안 될 때 — 가능 태그 + `[긴급]`**
- 예) `[티켓][긴급] 장애 대응`
- → 운영자 화면에 ⚠️로 표시되어, 가급적 제외하고 신중히 판단합니다.

> ※ 회의·미팅·태그 없는 일정은 기본 '가능'으로 봅니다. **빼면 안 되는 일(고객사 현장·외근·출장·휴가)은 반드시 🔴태그나 휴가로 표시**해 주세요!

### ■ 태그 다는 법 (자주 묻는 점)
- 반드시 **대괄호 [ ]** 사용 (소괄호 `(티켓)` 는 인식 안 됨)
- **태그 형식은 유연합니다** — 아래 모두 인식됩니다:
  `[티켓]` / `[티켓] 한국산업진흥원` / `[티켓_한국산업진흥원]` / `[완료/티켓]`
  → 대괄호 안이 `티켓·파트너·내부·교육·대기` 로 시작하면 OK
- `[내부업무]` 는 `[내부]` 와 똑같이 인식됩니다
- **하루종일 일정**이어도 🟢태그가 있으면 '가능'으로 잡힙니다

### ■ 운영자(파트너프로그램팀) 사용 흐름
1. 아웃룩 동기화 → 2. 라운드로빈 돌림판(날짜·시간 입력) → 3. 돌리기(재실행해도 동일인 = 공정) → 4. 배포지원 투입(담당자 메일·기록 자동)
- 애매한 일정은 클릭해 **가능/불가/긴급으로 직접 조정** 가능 (전 운영자에게 공유)
- 선정된 인원은 투입과 동시에 **후순위로 밀려** 공정하게 순환됩니다
- 당첨자 초기화가 필요하면 **'✕ 당첨자 초기화'** 버튼 사용 (이력은 DB에 기록)

자세한 규칙은 대시보드 상단 **'📖 사용가이드' / '🏷️ 태그 안내'** 에서 언제든 확인하실 수 있습니다.
